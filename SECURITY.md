## Security Policy

### Reporting a Vulnerability

If you believe you have discovered a security vulnerability in this project, please report it responsibly.

Please do **not** create a public GitHub issue for security vulnerabilities.

Instead, contact the maintainers directly:

solutions@trustthink.com

Include the following information in your report if possible:

\- Description of the vulnerability
\- Steps to reproduce the issue
\- Potential impact
\- Suggested remediation (if known)

We will acknowledge receipt of the report and work with you to address the issue.

### Scope

This repository provides a reference implementation for generating ISO/IEC 13888-3 non-repudiation evidence tokens.

Security issues may include:

\- Incorrect token encoding
\- Message imprint generation flaws
\- ASN.1 structure handling errors
\- Dependency vulnerabilities
\- Implementation errors affecting token integrity

### Responsible Disclosure

We request that security researchers provide a reasonable period for remediation before publicly disclosing vulnerabilities.
